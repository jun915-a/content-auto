# GitHub CLI Now Collects Pseudo-Anonymous Telemetry

*Insert header image here*

GitHub CLI is now collecting pseudo-anonymous telemetry data to improve the tool. Learn what data is collected and how to opt-out.

## 🔑 The Core of This Topic
The GitHub CLI, a powerful command-line tool for interacting with GitHub, has introduced telemetry collection. This means it will now send back usage data to GitHub. The data is anonymized and designed to help the GitHub team understand how the tool is used and identify areas for improvement.

## ⚡ 5-Second Key Points
- **Telemetry Enabled**: GitHub CLI now sends usage data.
- **Pseudo-Anonymous**: Data is anonymized, not personally identifiable.
- **Opt-Out Available**: Users can disable telemetry collection.

## 📈 Detailed Breakdown
**Data Collection**
The telemetry includes information about command usage, performance metrics, and errors encountered. It does not collect sensitive information like repository names, file contents, or authentication tokens. The goal is to understand which commands are popular, where users might be struggling, and if there are bugs.

**Anonymization Process**
GitHub emphasizes that the collected data is pseudo-anonymous. This means that while it tracks usage patterns, it's not linked back to individual users or their specific GitHub accounts. Efforts are made to ensure no personally identifiable information (PII) is transmitted.

> 💡 Insight: The primary driver for this change is to gain actionable insights for enhancing the developer experience with the GitHub CLI.

**Opting Out**
Users concerned about privacy or who simply don't wish to contribute telemetry data can easily opt out. This can be done via a configuration setting within the GitHub CLI itself, ensuring user control over data sharing.

## 🎯 Real-World Impact
- **Improved Tooling**: Insights help prioritize feature development and bug fixes.
- **Enhanced Performance**: Data can reveal performance bottlenecks to address.
- **Better Documentation**: Understanding common issues can lead to clearer help messages.

## ✨ Conclusion
While telemetry collection can raise privacy concerns, GitHub's approach with pseudo-anonymity and an easy opt-out aims to balance product improvement with user control. Developers can choose to contribute to the CLI's evolution or opt out entirely.
